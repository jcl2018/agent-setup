# Home Audit Checklist

- [ ] The global weekly audit state was checked in `~/.ai_shared/knowledge/progress-tracker.md`.
- [ ] The repo-local weekly audit state was checked in `.ai_shared/knowledge/progress-tracker.md` when the task is in a repo.
- [ ] Repo-local home-audit workflow, checklist, and audit-rule notes were consulted when the repo defines them.
- [ ] The `lv0-home-auditor` local-repo checklist and repo-audit helper were used when the audit scope included generic repo hygiene checks.
- [ ] Tracked local tracking files and tool runtime/session files that should stay off remotes were checked and recorded when the audit scope included a repo.
- [ ] User-specific absolute filesystem paths in tracked shareable files were checked and recorded when the audit scope included a repo.
- [ ] Shared knowledge, templates, workflows, checklists, examples, tasks, and canonical skill definitions are positioned in the expected folders.
- [ ] Tool-specific wrappers and entrypoint docs are still aligned with the canonical shared logic.
- [ ] The shared home folders needed by Codex and Claude were confirmed accessible.
- [ ] Outdated, dead, conflicted, or misplaced files and folders were identified as cleanup candidates.
- [ ] Repo-specific rules and continuity docs were checked for the repo that triggered the weekly pass.
- [ ] Managed repo-to-home discrepancies were checked and recorded when the repo's audit rules require mirror alignment.
- [ ] Cleanup actions that would delete or move content were not performed without explicit user approval.
- [ ] Home-level and repo-level progress/future-plan notes were updated with findings, validation, and follow-ups when those audit gates ran.

# Coding Standards

## General Expectations
- Prefer the smallest safe change that matches existing patterns.
- Keep naming, structure, and formatting consistent with the surrounding code.
- Avoid unrelated refactors unless the task explicitly calls for them.
- Add comments only when the intent would otherwise be hard to recover.

## Reliability
- Favor readable, testable code over clever shortcuts.
- Handle errors explicitly when the surrounding code does.
- Preserve backwards compatibility unless the task says otherwise.

## Validation
- Run targeted validation when possible.
- If validation cannot be run, say so clearly and explain why.

- Add project-specific rules here as they emerge.
